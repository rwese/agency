# Agency v2.0 - CLI Reference

## Command Overview

```
agency [--version] [--help]
agency <command> [<args>]
```

## Global Flags

| Flag | Description |
|------|-------------|
| `--version` | Show version |
| `--help` | Show help |
| `--json` | Output JSON (where supported) |

## Commands

### Project Management

#### `agency init-project`

Create a new project with tmux session, `.agency/` directory, and optional manager.

```bash
agency init-project --dir <path> [--template <name>] [--start-manager <name>] [--force]
```

| Flag | Required | Description |
|------|----------|-------------|
| `--dir` | Yes | Project directory path |
| `--template` | No | Template name (default: basic) or GitHub URL |
| `--start-manager` | No | Manager name to start immediately |
| `--force` | No | Overwrite existing session |

**Behavior:**
1. Find git root from `--dir`
2. Create tmux session `agency-<basename>` with socket `agency-<project>`
3. Copy template to `.agency/` (local templates first, then GitHub fallback)
4. If `--start-manager`, create manager window `[MGR] <name>` at index 0
5. Create `.agency/.scripts/` for launch scripts

**Examples:**
```bash
# Create project with default (basic) template
agency init-project --dir ~/projects/api

# Create with named template (api, fullstack)
agency init-project --dir ~/projects/api --template api

# Create with custom GitHub template
agency init-project --dir ~/projects/api --template https://github.com/user/repo/tree/main/my-template

# Create and start manager immediately
agency init-project --dir ~/projects/api --start-manager coordinator

# Recreate existing project
agency init-project --dir ~/projects/api --force
```

#### `agency start`

Start an agent in the project session.

```bash
agency start <name> --dir <path>
```

| Flag | Required | Description |
|------|----------|-------------|
| `name` | Yes | Agent name (must exist in `agents.yaml`) |
| `--dir` | Yes | Project directory |

**Behavior:**
1. Find git root from `--dir`
2. If session doesn't exist, create it (auto-init)
3. If agent name exists, attach to existing window
4. Otherwise, create new window at next index
5. Generate launch script in `.agency/.scripts/`
6. Launch agent with personality from config

**Examples:**
```bash
agency start coder --dir ~/projects/api
agency start tester --dir ~/projects/api
```

#### `agency stop`

Stop all windows in a session gracefully.

```bash
agency stop <session> [--timeout <seconds>]
```

| Flag | Required | Description |
|------|----------|-------------|
| `session` | Yes | Session name (without `agency-` prefix) |
| `--timeout` | No | Grace period before force kill (default: agent decides) |

**Behavior:**
1. Send shutdown message to all windows simultaneously
2. Wait for agents to finish (timeout per-agent)
3. Manager exits last
4. Kill tmux session
5. Clean up tmux socket

**Examples:**
```bash
agency stop api
agency stop api --timeout 60
```

#### `agency resume`

Resume a halted session.

```bash
agency resume <session>
```

| Flag | Required | Description |
|------|----------|-------------|
| `session` | Yes | Session name |

**Behavior:**
1. Check for `.halted` file
2. Rename session from `*-HALTED` back to normal
3. Restart manager with `AGENCY_RESUMING=true`
4. Manager reads `.halted`, processes pending tasks
5. Session continues from halt state

**Examples:**
```bash
agency resume api
```

#### `agency attach`

Attach to a tmux session.

```bash
agency attach <session>
```

| Flag | Required | Description |
|------|----------|-------------|
| `session` | Yes | Session name |

**Behavior:**
Uses standard tmux attach. Use tmux keybindings (`Ctrl-b d`) to detach.

**Examples:**
```bash
agency attach api
```

#### `agency list`

List all agency sessions.

```bash
agency list
```

**Output:**
```
agency-api
  [MGR] coordinator
  coder
  tester

agency-web
  [MGR] coordinator
  frontend
  backend
```

#### `agency init`

**DEPRECATED in v2.0**. Use `init-project`.

### Skill Management

#### `agency skill install`

Install the agency skill to a specified path.

```bash
agency skill install <path> [--force]
```

| Flag | Required | Description |
|------|----------|-------------|
| `path` | Yes | Target directory for skill installation |
| `--force` | No | Overwrite if skill already exists |

**Behavior:**
1. Copies the agency skill to `<path>/agency/`
2. Creates parent directories as needed
3. Fails if skill exists unless `--force` is used

**Examples:**
```bash
# Install to global skills directory
agency skill install ~/.pi/agent/skills/

# Install to project directory
agency skill install ~/projects/myproject/

# Overwrite existing
agency skill install ~/.pi/agent/skills/ --force
```

**Note:** Only available when `AGENCY_ROLE` environment variable is not set.

### Task Management

#### `agency tasks`

Task management commands. Run from project directory (or walks up to git root).

```bash
agency tasks <subcommand> [<args>]
```

#### `agency tasks list`

List all active tasks.

```bash
agency tasks list [--status <status>] [--assignee <agent>]
```

| Flag | Required | Description |
|------|----------|-------------|
| `--status` | No | Filter by status |
| `--assignee` | No | Filter by assigned agent |

**Output (Markdown):**
```markdown
## swift-bear-a3f2

- status: in_progress
- priority: normal
- assigned_to: coder
- description: Implement authentication API
- created_at: 2024-01-15T10:30:00Z
- started_at: 2024-01-15T10:35:00Z

## jade-owl-7f2a

- status: pending
- priority: high
- assigned_to: null
- description: Write unit tests for auth module
- created_at: 2024-01-15T10:40:00Z
```

#### `agency tasks add`

Create a new task.

```bash
agency tasks add -d <description> [-a <agent>] [-p <priority>]
```

| Flag | Required | Description |
|------|----------|-------------|
| `-d`, `--description` | Yes | Task description |
| `-a`, `--assignee` | No | Initial assignee (manager usually assigns) |
| `-p`, `--priority` | No | Priority: low, normal, high (default: low) |

**Behavior:**
1. Acquire filelock on `tasks.json`
2. Generate task ID: `word-word-hex`
3. Create `.agency/tasks/<id>/task.json`
4. Add task to `tasks.json`
5. Release lock

**Examples:**
```bash
agency tasks add -d "Implement user authentication"
agency tasks add -d "Fix login bug" -p high
agency tasks add -d "Write docs" -a coder
```

#### `agency tasks show`

Show task details.

```bash
agency tasks show <task_id>
```

**Output (Markdown):**
```markdown

## Task

- **Description**: Implement authentication API
- **Status**: in_progress
- **Priority**: normal
- **Assigned to**: coder
- **Created**: 2024-01-15T10:30:00Z
- **Started**: 2024-01-15T10:35:00Z

## Result

_None yet_
```

#### `agency tasks complete`

Mark task as complete (moves to pending approval).

```bash
agency tasks complete <task_id> --result <text> [--files <json>] [--diff <text>] [--summary <text>]
```

| Flag | Required | Description |
|------|----------|-------------|
| `task_id` | Yes | Task to complete |
| `--result` | Yes | Result summary (Markdown) |
| `--files` | No | JSON array of changed files |
| `--diff` | No | Git diff of changes |
| `--summary` | No | Summary for artifacts |

**Behavior:**
1. Create `result.json` in task directory
2. Update `tasks.json`: status → `pending_approval`
3. Move task data to `.agency/pending/<id>.json` (full task + result)
4. Manager polls for pending completions

**Examples:**
```bash
agency tasks complete swift-bear-a3f2 --result "Implemented JWT auth with refresh tokens"
agency tasks complete swift-bear-a3f2 --result "Fixed bug" --files '["src/auth.py"]'
```

#### `agency tasks assign`

Assign task to an agent.

```bash
agency tasks assign <task_id> <agent>
```

| Flag | Required | Description |
|------|----------|-------------|
| `task_id` | Yes | Task to assign |
| `agent` | Yes | Agent name |

**Behavior:**
1. Verify agent exists in session
2. Verify agent is free (no in_progress tasks)
3. Update `tasks.json`: assigned_to

**Examples:**
```bash
agency tasks assign swift-bear-a3f2 coder
```

#### `agency tasks update`

Update task fields.

```bash
agency tasks update <task_id> [--status <status>] [--priority <priority>]
```

| Flag | Required | Description |
|------|----------|-------------|
| `task_id` | Yes | Task to update |
| `--status` | No | New status |
| `--priority` | No | New priority |

**Examples:**
```bash
agency tasks update swift-bear-a3f2 --status pending
agency tasks update swift-bear-a3f2 --priority high
```

#### `agency tasks delete`

Delete a task.

```bash
agency tasks delete <task_id>
```

| Flag | Required | Description |
|------|----------|-------------|
| `task_id` | Yes | Task to delete |

**Behavior:**
1. Remove from `tasks.json`
2. Delete `.agency/tasks/<id>/` directory

**Examples:**
```bash
agency tasks delete swift-bear-a3f2
```

List completed tasks.

### Heartbeat Commands

Heartbeat processes monitor tasks and notify agents of available work. They run in the background.
Role is determined by `AGENCY_ROLE` env var (`MANAGER` or `AGENT`), agent name from `AGENCY_AGENT`.

#### `agency heartbeat start`

Start a heartbeat process.

```bash
agency heartbeat start
```

**Required env vars:**
- `AGENCY_ROLE=manager` or `AGENCY_ROLE=agent`
- For agents: `AGENCY_AGENT=<name>`

**Examples:**
```bash
# Start manager heartbeat
AGENCY_ROLE=manager agency heartbeat start

# Start agent heartbeat
AGENCY_ROLE=agent AGENCY_AGENT=backend agency heartbeat start
```

#### `agency heartbeat stop`

Stop heartbeat process.

```bash
agency heartbeat stop
```

- Agents: stops own heartbeat only
- Managers or no role: stops all heartbeats

#### `agency heartbeat status`

Show heartbeat status.

```bash
agency heartbeat status
```

**Output:**
```markdown
# Heartbeat Status

- **Manager**: 🟢 running (PID: 12345)

## Agents

- **backend**: 🟢 running (PID: 12346)
- **frontend**: ⚪ stopped
```


#### `agency heartbeat logs`

Show own heartbeat logs.

```bash
agency heartbeat logs [-n <lines>]
```

**Examples:**
```bash
agency heartbeat logs
agency heartbeat logs -n 50
```

**Heartbeat Behavior:**
- **Manager**: Notifies coordinator of unassigned tasks and pending approvals (only when nothing in progress)
- **Agent**: Notifies agent of assigned tasks (only when no task in progress)
- Notifications via pi-inject Unix socket (if available)

### Utility Commands

#### `agency version`

Show version.

```bash
agency version
```

#### `agency completions`

Print shell completion script.

```bash
agency completions <shell>
```

| Arg | Description |
|-----|-------------|
| `bash` | Bash completions |
| `zsh` | Zsh completions |
| `fish` | Fish completions |

**Examples:**
```bash
# Bash
source <(agency completions bash)

# Zsh
source <(agency completions zsh)
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General error |
| 2 | Usage error (invalid arguments) |
| 3 | Not found (session, task, etc.) |
| 4 | Conflict (already exists) |
| 5 | Lock error |

---

## Error Output

Errors are written to stderr with format:

```
[ERROR] <message>
```

With `--json` flag:
```json
{"error": "Session not found", "session": "api"}
```
