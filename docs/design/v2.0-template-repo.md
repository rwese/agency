# Agency v2.0 - Template Repository

## Overview

Agency uses GitHub repositories as templates for project initialization.

## Default Template Repository

**URL:** https://github.com/rwese/agency-templates

This repository contains starter templates for new Agency projects.

## Repository Structure

```

├── basic/

## Template Format

Each template is a directory containing a `.agency/` subdirectory. When a template is used, the `.agency/` directory is extracted to the project root.

### Template Requirements

```
<template-name>/
└── .agency/
    ├── config.yaml           # Required
    ├── manager.yaml          # Optional
    ├── agents.yaml           # Optional
    └── agents/              # Optional
        └── *.yaml
```

### config.yaml

```yaml
project: <template-name>
shell: bash
template_url: https://github.com/rwese/agency-templates
```

### manager.yaml

```yaml
personality: |
  You are the project coordinator.

  ## Your Role
  - Manage tasks using `agency tasks` commands
  - Assign work to free agents
  - Review completed work

poll_interval: 30
auto_approve: false
```

### agents.yaml

```yaml
agents:
  - name: coder
    config: agents/coder.yaml
```

### agents/<name>.yaml

```yaml
name: coder
personality: personality.md
```

## Creating a Template

### 1. Fork or Create Repository

```bash
# Clone the templates repo
gh repo clone rwese/agency-templates
cd agency-templates

# Create your template directory
mkdir -p my-template/.agency
```

### 2. Create Configuration Files

Create the necessary `.agency/` files:

```bash
# config.yaml
cat > my-template/.agency/config.yaml << 'EOF'
project: my-template
shell: bash
EOF

# manager.yaml
cat > my-template/.agency/manager.yaml << 'EOF'
personality: |
  You are the project coordinator.

poll_interval: 30
EOF

# agents.yaml
cat > my-template/.agency/agents.yaml << 'EOF'
agents:
  - name: developer
    config: agents/developer.yaml
EOF

# Create agent config directory
mkdir -p my-template/.agency/agents

cat > my-template/.agency/agents/developer.yaml << 'EOF'
name: developer
personality: personality.md
EOF

cat > my-template/.agency/agents/developer/personality.md << 'EOF'
# Developer Personality

You are a skilled developer focused on clean, maintainable code.

## Guidelines
- Write tests for all new code
- Follow project style guidelines
- Document your work
EOF

# README
cat > my-template/.agency/README.md << 'EOF'
# My Template

This is a custom Agency template.

## Usage

```bash
agency init-project --dir ~/projects/myproject \
  --template https://github.com/yourusername/agency-templates/tree/main/my-template
```
EOF
```

### 3. Push and Test

```bash
git add .
git commit -m "Add my-template"
git push

# Test with agency
agency init-project --dir /tmp/test-project \
  --template https://github.com/yourusername/agency-templates/tree/main/my-template
```

## Using Templates

### Basic Usage

```bash
# Using default template
agency init-project --dir ~/projects/api

# Using specific template
agency init-project --dir ~/projects/api \
  --template https://github.com/rwese/agency-templates/tree/main/api

# Using custom template
agency init-project --dir ~/projects/myapp \
  --template https://github.com/yourusername/your-repo/tree/main/basic
```

### URL Formats

Agency accepts several URL formats:

| Format | Example |
|--------|---------|
| Full GitHub URL | `https://github.com/user/repo/tree/main/template` |
| Tree path | `https://github.com/user/repo/tree/main/.agency-templates/basic` |
| Short owner/repo | `user/repo` (uses root) |
| Owner/repo:subdir | `user/repo:template` |

### Subdirectory Templates

For templates in subdirectories:

```bash
# Template at github.com/user/repo/tree/main/.agency-templates/basic
--template https://github.com/user/repo/tree/main/.agency-templates/basic

# Shorthand
--template https://github.com/user/repo/tree/main/basic
# (Agency looks for .agency/ in the specified path)
```

## Caching

Templates are cached in `~/.cache/agency/templates/`.

```
~/.cache/agency/templates/
├── github.com/
│   └── rwese/
│       └── agency-templates/
│           └── basic/
│               └── .agency/
└── github.com/
    └── yourusername/
        └── your-repo/
            └── template/
                └── .agency/
```

### Cache Management

```bash
# Clear specific template cache
rm -rf ~/.cache/agency/templates/github.com/user/repo

# Clear all template cache
rm -rf ~/.cache/agency/templates/

# Bypass cache (always download)
agency init-project --dir ~/projects/api --template <url> --refresh
```

## Best Practices

### 1. Version Your Templates

Use git tags for template versions:

```bash
# Create versioned template
git tag v1.0.0
git push origin v1.0.0

# Use specific version
--template https://github.com/user/repo/tree/v1.0.0/basic
```

### 2. Include Documentation

Always include a README in your template:

```markdown
# Template Name

Brief description of what this template is for.

## Pre-configured Agents

- `developer`: General purpose developer

## Usage

```bash
agency init-project --dir ~/projects/myproject \
  --template https://github.com/user/repo/tree/main/my-template
```
```

### 3. Test Your Templates

```bash
# Create test project
agency init-project --dir /tmp/test-template \
  --template https://github.com/yourusername/agency-templates/tree/main/your-template

# Verify structure
ls -la /tmp/test-template/.agency/

# Clean up
agency stop template-test
rm -rf /tmp/test-template
```

### 4. Use Meaningful Names

Template names should reflect their purpose:

| Good | Bad |
|------|-----|
| `python-api` | `template1` |
| `react-frontend` | `my-stuff` |
| `fullstack-django` | `default` |
