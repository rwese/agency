# Agency v2.0 - Design Specification

**Version:** 2.0.0
**Status:** Design Complete
**Last Updated:** 2024

## Overview

Agency v2.0 is a tmux-based AI agent orchestration tool that manages multi-agent development workflows through a **task-centric model**.

### Key Design Principles

1. **Tasks are the canonical source of truth** - All work state lives in tasks
2. **Agents start on-demand** - Only run agents when work is available
3. **Crash detection via PID** - Store process ID with task, verify existence
4. **Session persistence** - pi session IDs stored for restart after rejection
5. **Parallelism via config** - `parallel_limit` controls concurrent work

### Key Changes from v0.x

| Aspect | v0.x | v2.0 |
|--------|------|------|
| State Source | Session + Tasks | Tasks only |
| Agent Start | All at session start | On-demand |
| Crash Detection | Window polling | PID check |
| Session Persistence | None | pi session ID |
| Review | Manager only | Separate reviewer agent |

## Task Lifecycle

```
pending ──────► in_progress ──────► pending_approval ──────► completed
                     │                      │
                     │                      ▼
                     │                  rejected
                     │                      │
                     └──────────────────────┘
                            (restart)
```

### Task Fields

```python
@dataclass
class Task:
    task_id: str
    description: str
    status: str  # pending, in_progress, pending_approval, completed, failed
    priority: str  # low, normal, high
    assigned_to: str | None

    # Agent info (set when picked up)
    agent_info: dict | None = None
    # {
    #   "session_id": "pi-session-coder-a1b2c3",
    #   "pid": 12345,
    #   "started_at": "2024-01-15T10:30:00Z"
    # }

    # Review info
    rejection_reason: str | None
    review_notes: str | None

    depends_on: list[str] | None
```

## Crash Detection

When an agent picks up a task:
1. Task marked `in_progress`
2. Agent info stored (session_id, PID, timestamp)
3. Heartbeat periodically checks if PID exists
4. If PID dead → task reverts to `pending`

```python
def check_stale_tasks(agency_dir):
    for task in store.get_in_progress_tasks():
        if task.agent_info and not process_exists(task.agent_info.pid):
            store.clear_agent_info(task.task_id)
            # Task now available for reassignment
```

## Agent Lifecycle

```
┌────────────────────────────────────────────────────────────┐
│ AGENT START                                                │
│  1. Query: tasks assigned_to=me AND status=pending         │
│  2. Pick highest priority                                 │
│  3. Mark in_progress with PID                            │
│  4. Store session_id for restart                          │
└────────────────────────────────────────────────────────────┘
                         │
                         ▼
┌────────────────────────────────────────────────────────────┐
│ WORK                                                        │
│  1. Complete task → pending_approval                        │
│  2. Or rejection → failed                                  │
└────────────────────────────────────────────────────────────┘
                         │
                         ▼ (rejection)
┌────────────────────────────────────────────────────────────┐
│ RESTART                                                     │
│  1. Get session_id from task                              │
│  2. Restart with same session                             │
│  3. Inject fix prompt                                     │
└────────────────────────────────────────────────────────────┘
```

## Parallelism

```yaml
# config.yaml
parallel_limit: 2  # Per-agent task limit
```

- Each agent works on up to N tasks
- Manager tracks total busy count
- Agents start only when slots available

## Review Flow

```
Agent completes → pending_approval
                      │
                      ▼
           Reviewer starts (fresh context)
                      │
        ┌─────────────┴─────────────┐
        ▼                           ▼
   APPROVE                         REJECT
        │                           │
        ▼                           ▼
   completed              → restart agent with fix prompt
```

## Stop Modes

| Mode | Behavior |
|------|----------|
| Graceful | Finish current tasks, wrap-up, exit |
| Immediate | Interrupt work, send wrap-up, wait, exit |
| Force | Kill all processes immediately |

## Quick Reference

```bash
# Create project
agency init-project --dir ~/projects/api

# Session management
agency session start    # Start manager + configured agents
agency session stop    # Graceful shutdown
agency session attach  # Attach to session

# Task management
agency tasks add -d "Implement auth"
agency tasks assign swift-bear-a3f2 coder
agency tasks-agent complete swift-bear-a3f2 --result "Done"
agency tasks approve swift-bear-a3f2
agency tasks reject swift-bear-a3f2 --reason "Missing tests"
```
