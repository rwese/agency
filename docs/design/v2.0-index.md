# Agency v2.0 - Design Specification

**Version:** 2.0.0  
**Status:** Design Complete  
**Last Updated:** 2024

## Overview

Agency v2.0 is a tmux-based AI agent orchestration tool that manages multi-agent development workflows through a project-centric model.

### Key Changes from v0.x

| Aspect | v0.x | v2.0 |
|--------|------|------|
| Session Model | Agents in project sessions, managers separate | All in same project session |
| Task IDs | `TASK001` | `swift-bear-a3f2` |
| Task Storage | Single `tasks.json` | `tasks.json` + `.agency/tasks/<id>/` |
| Manager | `agency-manager-<name>` session | `[MGR] <name>` window at index 0 |
| TUI | Included | **Removed** |
| Templates | None | GitHub-based with caching |

## Quick Reference

### Commands

```bash
# Create project
agency init-project --dir ~/projects/api --start-manager coordinator

# Start agents
agency start coder --dir ~/projects/api
agency start tester --dir ~/projects/api

# Manage tasks
agency tasks add -d "Implement auth"
agency tasks assign swift-bear-a3f2 coder
agency tasks complete swift-bear-a3f2 --result "Done"
agency tasks list
agency tasks history

# Session management
agency stop api
agency resume api
agency attach api
agency list
```

### Task Statuses

```
pending ──────► in_progress ──────► pending_approval ──────► completed
                     │                      │
                     │                      ▼
                     │                  failed
                     │                      │
                     └──────────────────────┘
                            (retry)
```

### File Structure

```
.agency/
├── config.yaml         # Project settings
├── manager.yaml        # Manager personality + settings
├── agents.yaml         # Agent registry
├── tasks.json         # Active tasks (v2 schema)
├── .halted           # Halt marker
├── agents/
│   ├── coder.yaml
│   └── tester.yaml
├── tasks/
│   └── <task_id>/
│       ├── task.json
│       └── result.json
├── pending/
│   └── <task_id>.json  # Pending approval
└── .scripts/
    └── *.sh            # Launch scripts
```

## Breaking Changes Summary

1. **`init` removed** → Use `init-project`
2. **`start-manager`** → Creates window in project session, not separate session
3. **`stop-manager` removed** → Use `stop`
4. **TUI removed** → CLI only
5. **Task IDs** → `word-word-xxxx` format
6. **Config location** → All project-local in `.agency/`
7. **Global config deprecated** → Templates in `~/.cache/agency/templates/`

---

## Questions Resolved

See individual design documents for complete list. Key decisions:

- ✓ Manager placement: Window in project session at index 0
- ✓ Task assignment: 1 active task per agent, queue allowed
- ✓ Task archival: Task moved to `.agency/tasks/<id>/`
- ✓ Halt handling: `.halted` file + session rename
- ✓ Resume: Manager reads `.halted`, processes pending tasks
- ✓ Template system: GitHub-based with caching
- ✓ Locking: filelock for tasks.json, atomic rename for pending/
