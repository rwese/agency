# Agency v2.0 - Entity Relationships

## Overview

Agency v2.0 introduces a project-centric model where a **Project** contains exactly one **Manager** and multiple **Agents**, all orchestrated through **Tasks**.

## Relationships

| From | To | Cardinality | Description |
|------|----|-------------|-------------|
| Project | Manager | 1:1 | One manager per project |
| Project | Agent | 1:N | Multiple agents per project |
| Manager | Task | 1:N | Manager creates and assigns tasks |
| Agent | Task | 1:1 | Agent works on one active task |

## Entity Definitions

### Project

A **Project** is a tmux session that encapsulates a complete development workspace.

```
tmux session: agency-<project-name>
tmux socket:  agency-<project-name>
working dir:  <project-root>
```

- **Session Name**: Derived from directory basename (e.g., `~/projects/api` → `agency-api`)
- **Socket**: Per-project isolation for tmux server
- **Git Root**: `.agency/` directory lives at git repository root

### Manager

A **Manager** is the orchestrator agent that coordinates task assignment and monitors agent health.

```
window name:  [MGR] <manager-name>
window index: 0
badge:        [MGR] prefix + colored status bar
config:       .agency/manager.yaml
```

- **Single Instance**: Only one manager per project
- **Task Broker**: Creates tasks, assigns to free agents
- **Health Monitor**: Polls tmux for agent status
- **Approval Gate**: Reviews pending task completions

### Agent

An **Agent** is a worker that processes tasks assigned by the manager.

```
window name:  <agent-name>
window index: 1+
config:       .agency/agents/<name>.yaml
personality:  personality.md (referenced from config)
```

- **Idle Until Assigned**: Waits for task assignment
- **Single Active Task**: Only one task in `in_progress` status
- **Queue Allowed**: Can have pending tasks queued
- **Tool Access**: Via shell environment (not in config)

### Task

A **Task** represents a unit of work to be completed by an agent.

```
task_id:      swift-bear-a3f2  (word-word-hex format)
status:       pending | in_progress | pending_approval | completed | failed
priority:     low | normal | high
assigned_to:  <agent-name> | null
```

- **UUID-based ID**: Uses eff.org wordlist + hex suffix
- **Lifecycle States**: pending → in_progress → pending_approval → completed/failed
- **Result Storage**: Archived to `.agency/tasks/<id>/` on completion
- **1:1 Active**: Agent has at most one `in_progress` task

## Task State Machine

```
                    ┌──────────────┐
                    │   pending    │ ← task created
                    └──────┬───────┘
                           │ manager assigns to free agent
                           ▼
                    ┌──────────────┐
         ┌─────────│ in_progress  │ ← agent working
         │         └──────┬───────┘
         │                │ agent completes
         │                ▼
         │         ┌──────────────┐
         │         │ pending_     │ ← awaiting manager approval
         │         │ approval     │
         │         └──────┬───────┘
         │                │
         │    ┌───────────┴───────────┐
         │    │                       │
         │    ▼                       ▼
         │ ┌──────────┐       ┌──────────┐
         │ │completed │       │  failed  │
         │ └──────────┘       └──────────┘
         │    │                       │
         │    │                       │ manager may recreate
         │    │                       ▼
         │    │               ┌──────────┐
         │    └──────────────►│ pending  │
         │                    └──────────┘
         │                       ▲
         └───── agent retries   │
              (reject flow)     │
                                 │
              ┌──────────────────┘
              │ manager revokes assignment
              ▼
         ┌──────────┐
         │  failed  │ → assigned_to cleared
         └──────────┘   back to unassigned queue
```

## Implementation Notes

### Session Naming

| Entity | Pattern | Example |
|--------|---------|---------|
| Project | `agency-<basename>` | `agency-api` |
| Manager Window | `[MGR] <name>` | `[MGR] coordinator` |
| Agent Window | `<name>` | `coder` |

### Window Index Convention

- `0`: Manager (always)
- `1+`: Agents (in creation order)

### Free Agent Detection

An agent is **free** when:
- Has no tasks with `status == "in_progress"`
- Has no tasks with `status == "pending"` assigned to them

Free agents can receive new task assignments.
