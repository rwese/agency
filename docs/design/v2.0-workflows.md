# Agency v2.0 - Workflows

## Project Lifecycle

### Creating a New Project

```bash
# 1. Create project with default template
agency init-project --dir ~/projects/api

# 2. Start manager
agency start coordinator --dir ~/projects/api

# 3. Start agents
agency start coder --dir ~/projects/api
agency start tester --dir ~/projects/api

# 4. Attach and interact
agency attach api
```

**What happens:**

1. **Session Created**
   - tmux session: `agency-api`
   - tmux socket: `agency-api`
   - Working directory: `~/projects/api`

2. **`.agency/` Created**
   ```
   .agency/
   ├── config.yaml
   ├── manager.yaml
   ├── agents.yaml
   ├── README.md
   ├── agents/
   │   └── coder.yaml
   ├── tasks/
   ├── pending/
   └── .scripts/
   ```

3. **Manager Window Created**
   - Window: `[MGR] coordinator` at index 0
   - Badge: `[MGR]` prefix + colored status bar

4. **Agent Windows Created**
   - Window: `coder` at index 1
   - Window: `tester` at index 2

## Task Lifecycle

### Creating a Task

```bash
agency tasks add -d "Implement user authentication API"
```

**Flow:**

```
┌─────────────────┐
│  User/Manager   │
│  tasks add      │
└────────┬────────┘
         │
         │ 1. Create .agency/tasks/<id>/task.json
         ▼
┌─────────────────┐
│  .agency/tasks/ │
│  <id>/task.json │
└────────┬────────┘
         │
         │ 2. Add to tasks.json (with filelock)
         ▼
┌─────────────────┐
│  tasks.json     │
│  status=pending │
└────────┬────────┘
         │
         ▼
    ┌─────────┐
    │ pending │
    └─────────┘
```

**Result:**
```json
{
  "task_id": "swift-bear-a3f2",
  "description": "Implement user authentication API",
  "status": "pending",
  "priority": "low",
  "assigned_to": null,
  ...
}
```

### Assigning a Task

```bash
agency tasks assign swift-bear-a3f2 coder
```

**Flow:**

```
┌─────────────────┐     ┌─────────────────┐
│  Manager        │     │  tasks.json     │
│  Checks free    │────►│  assigned_to:   │
│  agents         │     │    coder        │
└─────────────────┘     └─────────────────┘
         │
         │ 3. Send task to agent via tmux
         ▼
┌─────────────────┐
│  Agent: coder   │
│  Receives task  │
└─────────────────┘
```

**Manager Logic:**
1. `agency tasks list` → see unassigned tasks
2. `agency list` → see available agents
3. Check agent is free (no in_progress tasks)
4. `agency tasks assign <id> coder`
5. Send message to coder via tmux

### Completing a Task

```bash
agency tasks complete swift-bear-a3f2 \
  --result "Implemented JWT auth with refresh tokens" \
  --files '["src/auth.py", "tests/test_auth.py"]'
```

**Flow:**

```
┌─────────────────┐
│  Agent: coder   │
│  tasks complete │
└────────┬────────┘
         │
         │ 1. Create result.json
         ▼
┌─────────────────────────────────┐
│  .agency/tasks/swift-bear-a3f2/ │
│  result.json                     │
└────────┬────────────────────────┘
         │
         │ 2. Update tasks.json (filelock)
         │    status: pending_approval
         ▼
┌─────────────────┐
│  tasks.json     │
│  status:        │
│  pending_approval│
└────────┬────────┘
         │
         │ 3. Move to pending/
         ▼
┌─────────────────────────────────┐
│  .agency/pending/               │
│  swift-bear-a3f2.json           │
│  (full task + result)            │
└─────────────────────────────────┘
```

### Manager Approval

```
┌─────────────────┐     ┌─────────────────┐
│  Manager        │     │  .agency/pending │
│  Polls pending/ │◄────│  *.json files   │
└────────┬────────┘     └─────────────────┘
         │
         │ 1. Read pending file
         │ 2. Review result
         │ 3. Decision: approve or reject
         ▼
    ┌─────────────────┐
    │  Approve?       │
    └────────┬────────┘
             │
     ┌───────┴───────┐
     │               │
     ▼               ▼
┌─────────┐   ┌──────────┐
│ Approve │   │ Reject   │
└────┬────┘   └────┬─────┘
     │              │
     │ 1. mv to    │ 1. Write
     │    tasks/    │    .rejected
     │              ▼
     │         ┌──────────┐
     │         │ Manager  │
     │         │ Sends    │
     │         │ feedback │
     │         └──────────┘
     ▼
┌─────────────────────────────────┐
│  .agency/tasks/swift-bear-a3f2/ │
│  ├── task.json                  │
│  └── result.json                │
└─────────────────────────────────┘
```

**Manager Commands:**
```bash
# Review pending
cat .agency/pending/swift-bear-a3f2.json

# Approve (archive)
mv .agency/pending/swift-bear-a3f2.json .agency/tasks/swift-bear-a3f2/
# Or use CLI
agency tasks approve swift-bear-a3f2

# Reject
echo "# Task Rejected" > .agency/pending/swift-bear-a3f2.rejected
# Manager sends feedback to agent
```

### Automated Review Flow (v2.0)

When a task is completed, the manager heartbeat automatically spawns a reviewer agent:

```
┌─────────────────┐     ┌─────────────────┐
│  Heartbeat      │────►│  Task moves to  │
│  Detects        │     │  pending_approval│
└────────┬────────┘     └────────┬────────┘
         │                      │
         │ 1. Check for         │
         │    orphaned tasks     │
         ▼                      │
┌─────────────────┐             │
│  Spawn reviewer │◄────────────┘
│  agent           │
└────────┬────────┘
         │
         │ 2. Reviewer reviews
         │    task result
         ▼
    ┌─────────────┐
    │  Approve?  │
    └──────┬──────┘
           │
    ┌─────┴─────┐
    ▼           ▼
┌────────┐ ┌────────┐
│Approve │ │Reject  │
└───┬────┘ └───┬────┘
    │           │
    │ 1. Move  │ 1. Set
    │    to    │    status
    │    done  │    to "failed"
    │           │    + feedback
    ▼           ▼
┌──────────┐ ┌──────────┐
│ Completed│ │ Restart  │
│          │ │ with     │
│          │ │ feedback │
└──────────┘ └──────────┘
```

**Key behaviors:**
- Reviewer is only spawned when task reaches `pending_approval`
- `reviewer_assigned` field tracks which reviewer is handling the task
- Orphaned tasks (no reviewer) are auto-detected and recovered
- Reviewer can approve (`agency tasks approve`) or reject (`agency tasks reject`)

### Lazy Agent Spawning (v2.0)

Agents are spawned **only when work is assigned to them**:

```
┌─────────────────────────────────────────────┐
│  Manager Heartbeat Loop                     │
└─────────────────────────────────────────────┘
                      │
         ┌────────────┼────────────┐
         ▼            ▼            ▼
    ┌─────────┐ ┌──────────┐ ┌───────────┐
    │ Assign  │ │ Spawn    │ │ Review    │
    │ Tasks   │ │ Agents   │ │ Pending   │
    └────┬────┘ └────┬─────┘ └─────┬─────┘
         │            │             │
         │ 1. Find    │             │
         │    unassign│             │
         │    ed tasks│             │
         │            │             │
         ▼            ▼             ▼
    ┌─────────────────────────────────────────┐
    │  Slot Coordination                      │
    │  - signals/slots-available.json        │
    │  - Agents wait for slot if at limit    │
    └─────────────────────────────────────────┘
```

**Spawning rules:**
1. Task must be **assigned** to an agent
2. Agent must **not already be running**
3. Agent must have **available slot** (capacity < parallel_limit)

**Slot signaling:**
- Slots tracked in `.agency/signals/slots-available.json`
- When task completes → slot released → waiting agents notified
- Supports blocking wait (`wait_for_slot()`)

## Session Management

### Normal Shutdown

```bash
agency stop api
```

**Flow:**

```
┌─────────────────────────────────────────────────────────┐
│  agency stop api                                        │
└────────────────────────┬────────────────────────────────┘
                         │
                         │ 1. Send shutdown to ALL windows
                         ▼
              ┌──────────────────────────┐
              │ [MGR] coordinator       │
              │ coder                   │
              │ tester                  │
              └────────────┬────────────┘
                           │
                           │ 2. Simultaneous graceful shutdown
                           │    - All agents finish work
                           │    - Archive pending tasks
                           │    - Exit
                           ▼
              ┌──────────────────────────┐
              │ Manager exits last      │
              └────────────┬────────────┘
                           │
                           │ 3. Kill tmux session
                           ▼
              ┌──────────────────────────┐
              │ Delete tmux socket     │
              │ agency-api             │
              └──────────────────────────┘
```

---

### Halt (Manager Crash)

```
┌─────────────────┐
│  Manager dies   │
│  unexpectedly   │
└────────┬────────┘
         │
         │ 1. Create .halted
         ▼
┌─────────────────┐
│  .agency/.halted │
│  {halted_at}    │
└────────┬────────┘
         │
         │ 2. Rename session/window
         ▼
┌─────────────────────────┐
│  Session: agency-api-   │
│    HALTED               │
│  Window: [HALTED] coord │
└─────────────────────────┘
         │
         ▼
    ┌──────────┐
    │ Agents   │
    │ continue │
    │ work     │
    └──────────┘
```

---

### Resume from Halt

```bash
agency resume api
```

**Flow:**

```
┌─────────────────┐
│  agency resume  │
└────────┬────────┘
         │
         │ 1. Check .halted exists
         ▼
┌─────────────────┐
│  .halted found  │
└────────┬────────┘
         │
         │ 2. Rename session
         │    agency-api-HALTED
         │    → agency-api
         ▼
┌─────────────────┐
│  Restart manager │
│  AGENCY_     │
│  RESUMING=true  │
└────────┬────────┘
         │
         │ 3. Manager reads .halted
         ▼
┌─────────────────┐
│  Process pending│
│  tasks         │
└────────┬────────┘
         │
         │ 4. .halted kept
         │    (manager clears)
         ▼
    ┌──────────┐
    │ Resuming │
    │ normal   │
    │ ops      │
    └──────────┘
```

## Template System

### Using a Template

```bash
agency init-project \
  --dir ~/projects/api \
  --template api
```

**Available templates:** `basic`, `api`, `fullstack`

**Flow:**

```
┌─────────────────────────┐
│  init --template <name> │
└────────────┬────────────┘
             │
             │ 1. Check local templates/
             │    (bundled with agency)
             ▼
┌─────────────────────────┐
│  Local found:           │
│  Copy to cache          │
│  ~/.cache/agency/...    │
└────────────┬────────────┘
             │
             │ (Local not found)
             │
             ▼
┌─────────────────────────┐
│  GitHub fallback:       │
│  Download archive       │
│  (--template <url>)     │
└────────────┬────────────┘
             │
             │ 2. Extract .agency/
             ▼
┌─────────────────────────┐
│  Copy to project:       │
│  project/.agency/       │
└────────────┬────────────┘
             │
             ▼
    ┌──────────────┐
    │ Continue     │
    │ init         │
    └──────────────┘
```

## Agent Discovery

### Manager Polls for Agents

```bash
# In manager's shell
agency list
```

**Output:**
```
agency-api
  [MGR] coordinator
  coder
  tester
```

### Checking Agent Status

```bash
# Manager checks if agent is free
agency tasks list --assignee coder
```

**Logic for free agent:**
1. No tasks with `status == "in_progress"`
2. No tasks with `status == "pending"` assigned to them

## Error Handling

### Session Not Found

```bash
$ agency attach api
[ERROR] Session not found: agency-api
Run 'agency init-project --dir <path>' first
```

### Task Not Found

```bash
$ agency tasks show invalid-id
[ERROR] Task not found: invalid-id
```

### Agent Already Exists

```bash
$ agency start coder --dir ~/projects/api
[INFO] Agent 'coder' already running, attaching...
# OR
[ERROR] Agent 'coder' already exists in session
Use 'agency attach api' to attach
```

### Lock Timeout

```bash
$ agency tasks add -d "..."
[ERROR] Could not acquire lock on tasks.json
Another process may be modifying tasks
Retry in a moment
```
