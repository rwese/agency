# Agency v2.0 - Data Models & Schemas

## Overview

This document defines the JSON/YAML schemas for all data structures in Agency v2.0.

## Task System

### tasks.json (v2)

The main task registry, stored at `.agency/tasks.json`.

```json
{
  "version": 2,
  "tasks": {
    "<task_id>": {
      "task_id": "string",
      "description": "string",
      "status": "pending|in_progress|pending_approval|completed|failed",
      "priority": "low|normal|high",
      "assigned_to": "string|null",
      "created_at": "ISO8601",
      "started_at": "ISO8601|null",
      "completed_at": "ISO8601|null",
      "result": "string|null",
      "result_path": "string|null"
    }
  }
}
```

**Fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `version` | integer | Yes | Schema version (always 2) |
| `tasks` | object | Yes | Map of task_id → task |
| `task_id` | string | Yes | Unique identifier (word-word-hex) |
| `description` | string | Yes | Human-readable task description |
| `status` | enum | Yes | Current task state |
| `priority` | enum | Yes | Task priority (default: low) |
| `assigned_to` | string\|null | Yes | Agent name or null |
| `created_at` | ISO8601 | Yes | Creation timestamp |
| `started_at` | ISO8601\|null | Yes | When work began |
| `completed_at` | ISO8601\|null | Yes | When completed |
| `result` | string\|null | Yes | Markdown result summary |
| `result_path` | string\|null | Yes | Path to result.json |

**Example:**
```json
{
  "version": 2,
  "tasks": {
    "swift-bear-a3f2": {
      "task_id": "swift-bear-a3f2",
      "description": "Implement authentication API",
      "status": "in_progress",
      "priority": "normal",
      "assigned_to": "coder",
      "created_at": "2024-01-15T10:30:00Z",
      "started_at": "2024-01-15T10:35:00Z",
      "completed_at": null,
      "result": null,
      "result_path": null
    }
  }
}
```

---

### Task Directory

For each completed task, a directory is created at `.agency/tasks/<task_id>/`.

```
.agency/tasks/
├── swift-bear-a3f2/
│   ├── task.json      # Full task definition
│   └── result.json    # Completion result
└── jade-owl-7f2a/
    ├── task.json
    └── result.json
```

---

### task.json (in task directory)

The complete task definition, stored for historical reference.

```json
{
  "task_id": "swift-bear-a3f2",
  "description": "Implement authentication API",
  "status": "completed",
  "priority": "normal",
  "assigned_to": "coder",
  "created_at": "2024-01-15T10:30:00Z",
  "started_at": "2024-01-15T10:35:00Z",
  "completed_at": "2024-01-15T14:30:00Z",
  "result": "Implemented JWT auth with refresh tokens",
  "result_path": ".agency/tasks/swift-bear-a3f2/result.json"
}
```

---

### result.json

The completion result with artifacts.

```json
{
  "result": "Implemented JWT authentication with JWT access tokens and refresh tokens. Added /auth/login and /auth/refresh endpoints.",
  "artifacts": {
    "files": ["src/auth.py", "src/middleware.py", "tests/test_auth.py"],
    "diff": "--- a/src/auth.py\n+++ b/src/auth.py\n@@ -1,3 +1,10 @@...",
    "summary": "Added authentication endpoints with JWT tokens"
  },
  "completed_at": "2024-01-15T14:30:00Z",
  "completed_by": "coder"
}
```

**Fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `result` | string | Yes | Markdown result summary |
| `artifacts` | object | No | Changed files and diffs |
| `artifacts.files` | array | No | List of modified file paths |
| `artifacts.diff` | string | No | Git diff of changes |
| `artifacts.summary` | string | No | Brief summary of changes |
| `completed_at` | ISO8601 | Yes | Completion timestamp |
| `completed_by` | string | No | Agent who completed |

### Pending Completion

When an agent completes a task, it's moved to `.agency/pending/`.

```
.agency/pending/
├── swift-bear-a3f2.json
└── jade-owl-7f2a.json
```

**Format:** Same as task.json + result.json combined.

```json
{
  "task_id": "swift-bear-a3f2",
  "description": "Implement authentication API",
  "status": "pending_approval",
  "priority": "normal",
  "assigned_to": "coder",
  "created_at": "2024-01-15T10:30:00Z",
  "started_at": "2024-01-15T10:35:00Z",
  "completed_at": "2024-01-15T14:30:00Z",
  "result": "Implemented JWT auth...",
  "result_path": ".agency/tasks/swift-bear-a3f2/result.json",
  "pending_approval_at": "2024-01-15T14:30:00Z"
}
```

---

### Rejection File

When a manager rejects a completion, a rejection file is created.

**Path:** `.agency/pending/<task_id>.rejected`

**Format:** Markdown

```markdown
# Task Rejected: swift-bear-a3f2

## Reason

The implementation is missing proper error handling for invalid tokens.

## Suggestions

1. Add try/catch around JWT decoding
2. Return 401 with appropriate error message
3. Add tests for invalid token scenarios

## Retry Count

1
```

## Configuration Files

### .agency/config.yaml

Project-level configuration.

```yaml
project: api
shell: bash
template_url: https://github.com/rwese/agency-templates
stop_timeout: 30
```

**Fields:**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `project` | string | Yes | - | Project name (derived from dir) |
| `shell` | string | No | bash | Default shell for agents |
| `template_url` | string | No | agency-templates | Default template repo |
| `stop_timeout` | integer | No | 30 | Grace period in seconds |

---

### .agency/manager.yaml

Manager configuration.

```yaml
personality: |
  You are the project coordinator for this team.

  ## Your Role
  You manage tasks and delegate work to agents.

  ## Task Management
  - Use `agency tasks list` to see pending tasks
  - Use `agency tasks assign <id> <agent>` to assign
  - Review completed tasks via `agency tasks history`

  ## Workflow
  1. Review incoming requests
  2. Create tasks with `agency tasks add -d "..."`
  3. Assign to free agents
  4. Review completions and archive

poll_interval: 30
auto_approve: false
max_retries: null
```

**Fields:**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `personality` | string | Yes | - | Manager system prompt |
| `poll_interval` | integer | No | 30 | Seconds between health checks |
| `auto_approve` | boolean | No | false | Auto-approve completions |
| `max_retries` | integer\|null | No | null | Max retries on reject (null=unlimited) |

---

### .agency/agents.yaml

Agent registry.

```yaml
agents:
  - name: coder
    config: agents/coder.yaml
  - name: tester
    config: agents/tester.yaml
```

**Fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agents` | array | Yes | List of agent definitions |
| `agents[].name` | string | Yes | Agent identifier |
| `agents[].config` | string | No | Path to config file (relative to .agency/) |

---

### .agency/agents/<name>.yaml

Individual agent configuration.

```yaml
name: coder
personality: personality.md
```

**Fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Agent identifier |
| `personality` | string | No | Path to personality file or inline |

**Note:** `working_dir` is derived from the project root. `tools` are configured in the shell environment, not here.

## Task ID Generation

Task IDs use the [eff.org short wordlist](https://www.eff.org/deeplinks/2016/02/announcing-eff-short-wordlist) with a 4-character hex suffix.

**Format:** `word-word-xxxx`

**Example:** `swift-bear-a3f2`

**Generation Algorithm:**
1. Pick 2 random words from wordlist
2. Generate 4 random hex characters
3. Format as `word-word-xxxx`

**Collision Handling:**
- Check if ID exists in tasks.json
- If collision, regenerate (extremely rare)

## Locking Strategy

### tasks.json

Uses `filelock.FileLock` for exclusive access.

```python
lock = FileLock(".agency/.tasks.lock")
with lock:
    # Read tasks.json
    # Modify tasks
    # Write tasks.json
```

**Lock file:** `.agency/.tasks.lock`

---

### pending/*.json

No locking required. Uses atomic operations:

1. Write to `.agency/pending/<id>.tmp`
2. `os.rename()` to `.agency/pending/<id>.json`

POSIX `rename()` is atomic and fails if target exists.
